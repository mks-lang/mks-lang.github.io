## 2024-04-28 - Documentation Integrity and Safety Communication
**Vulnerability:** Outdated or erroneous code examples in experimental documentation can lead to insecure user implementations or system instability.
**Learning:** Maintaining documentation parity in multi-language projects is critical to ensure safety warnings reach all users. A centralized "Reworking" UI pattern prevents users from relying on half-finished or deprecated features.
**Prevention:** Always inject prominent safety notices on documentation pages when the language is in an unstable/experimental state. Use minimalist placeholders for pages undergoing major redesign to avoid confusion.
